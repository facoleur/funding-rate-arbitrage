# AGENTS.md — backend/

Python backend for the option-arbitrage system. FastAPI + uv + asyncio + httpx + websockets + SQLModel + Alembic + Postgres.

## Layout

```

│   ├── health.py
│   ├── opportunities.py     # list + detail
```

## Entry-points (per container)

| Command | Container | Purpose |
|---|---|---|
| `uvicorn option_arb.main:app` | `api` | REST + SSE, read-only |
| `python -m option_arb.worker` | `workers` | WS + screener + alerter + rebalancer |
| `python -m option_arb.services.executor` | `executor` | state machine, poll PENDING opps |
| `python -m option_arb.backtest --file …` | ad-hoc | replay recorded books |
| `python -m option_arb.record --exchange … --duration …` | ad-hoc | record book snapshots |

## Config resolution

`option_arb.config.Settings` reads env (`.env` at project root). `load_config()` merges with `config.yaml` at `CONFIG_PATH` (default: `config.yaml`).

Env owns secrets + paths:
- `DATABASE_URL` — async URL for the app (`postgresql+asyncpg://…` in prod, `sqlite+aiosqlite://…` in tests).
- `ALEMBIC_DATABASE_URL` — sync URL for Alembic. Auto-derived from `DATABASE_URL` if unset (swaps `+asyncpg` → `+psycopg2`, drops `+aiosqlite`).
- `CONFIG_PATH`
- Exchange credentials (see auth section below)

YAML owns runtime knobs (thresholds, executor mode, kill-switches, per-exchange rate limits + REST/WS URLs).

## DB conventions

- **Postgres 16** in production (`postgres` service in `docker-compose.yml`, volume `postgres_data`).
- **SQLite** only in pytest (fresh temp file per test via `test_db` fixture in `tests/conftest.py`). Same SQLModel code — the migration `b5c114583a03_initial_schema.py` uses portable types.
- SQLite auto-enables WAL + `busy_timeout=5000` on connect (see `db/session.py::_configure_connection`). Postgres needs no equivalent.
- All writes owned by backend. Frontend read-only via REST.
- Alembic revisions in `db/migrations/versions/`. New revision: `make migrate-new msg="..."`.
- `Mode` enum on every opportunity/trade: `live | paper | backtest`. Same code path, only the injected exchange adapter differs.

## AbstractExchange contract

Every adapter (real or mock) implements:

```python
async list_instruments(underlying, max_expiries_ahead) -> list[Instrument]
async get_orderbook_l2(instrument) -> Book
def   ws_channels(instruments) -> list[str]
def   parse_ws_message(raw) -> TickerUpdate | None
async place_order(order) -> OrderResult          # requires Authenticator
async cancel_order(exchange_order_id) -> bool    # requires Authenticator
async get_balance_usd() -> Decimal               # requires Authenticator
async get_positions() -> list[dict]              # requires Authenticator
```

Every adapter MUST emit `Instrument.normalized_name = {UNDERLYING}-{YYYYMMDD}-{STRIKE}-{C|P}`. Deribit prices come in underlying units → the adapter multiplies by `underlying_price` to convert to USD.

Every adapter takes an optional `Authenticator`. Without one → private paths return `REJECTED` / empty results. Never hit the network with unauth private calls.

## Auth layer

Common ABC `Authenticator` (`exchanges/auth.py`) with three methods:
- `sign_rest(method, path, body) -> RestSignature` — headers/params/body_extra to attach.
- `authenticate_ws(ws)` — handshake sent right after WS connect.
- `sign_ws_message(msg)` — inject token/signature into outgoing WS message.

Concrete implementations:
- `NoAuth` — public-only. Raises `AuthNotReady` on any private call.
- `DeribitOAuth` — OAuth 2.0 `client_credentials`. Fetches token via `/public/auth` (bound at construction via `bind_auth_call`), caches until 60s before expiry, refresh under `asyncio.Lock`. Deribit's actual TTL ≈ 3600s.
- `DeriveAuth` (in `exchanges/derive_auth.py`) — specialized for Derive:
  - Wraps the official `derive-action-signing` package (`SignedAction` + `TradeModuleData`).
  - `sign_trade_action(...)` builds the action payload, signs it with the session key, calls `validate_signature()` as a self-check, returns the JSON dict to merge into `/private/order`.
  - `sign_rest(...)` produces the three `X-LYRAWALLET` / `X-LYRATIMESTAMP` / `X-LYRASIGNATURE` headers required on every REST `/private/*` call.
  - `ws_login_params()` returns the params for the WS `public/login` message if we ever route private orders over WS (currently REST-only).
  - Constants (DOMAIN_SEPARATOR, ACTION_TYPEHASH, TRADE_MODULE, WITHDRAW_MODULE, USDC_ASSET) live in `exchanges/derive_constants.py` for both `mainnet` (chain_id=957) and `testnet` (chain_id=901). Source: docs.derive.xyz/docs/protocol-constants.
- Aevo — currently `NoAuth`. Signing pattern to be added later.

`build_authenticator(exchange, settings, network)` picks the right class. Called by `registry.build_exchanges` with `network` taken from `ExchangeConfig.network` (defaults to `"testnet"` in `config.yaml`).

## HTTP rate limiting

`RestClient` (one per exchange) provides:
- **aiolimiter leaky bucket** with a reserved `priority` lane for the executor (default 5 req/s) so it can't be starved by metadata refreshes.
- **Retry** on 429 / 5xx with exponential backoff, honouring `Retry-After` headers.
- **Circuit breaker**: opens after 5 failures in 30s, half-open after 30s cooldown. While open, `request()` raises `CircuitOpenError`.

## Executor state machine (highest-blast-radius)

Loop every `executor.poll_interval_ms` (default 200ms):

```
opportunity(PENDING)
  ├─ kill-switch check (4 gates) → REJECTED w/ reason on trip
  ├─ fresh REST L2 refetch (parallel, both venues, timeout 500ms)
  │    on timeout/error → REJECTED (stale_book)
  ├─ walk book + binary-search size s.t. APR ≥ min_apr and notional ≤ cap
  │    if size < min_notional → REJECTED (size_too_small)
  │    if no size passes APR → REJECTED (apr_dropped)
  ├─ APPROVED + create Trade(PLACING) + 2 Order(PLACING) rows
  ├─ asyncio.gather(place_order(BUY IOC), place_order(SELL IOC))
  └─ outcome dispatch:
       both filled  → FILLED  (compute pnl)
       one filled   → HEDGING → market-out (IOC ±5%) → HEDGED or STUCK
       none filled  → FAILED
```

Kill-switches (4, all active):
1. **Manual file** `data/EXECUTOR_DISABLED` (`POST /api/executor/kill` creates it).
2. **Max open positions** = `count(trades WHERE status IN active)`.
3. **Max daily loss** = `SUM(net_pnl_usd WHERE opened_at >= today_utc)`.
4. **Max notional per trade** enforced inside `_walk_and_verify`.

**Every state transition persists to `trades` + `orders` before the next await.**

## Screener → Executor coupling

Zero direct in-process calls. Screener writes `Opportunity(status=PENDING)` in DB; executor polls `PENDING` every 200ms. Full audit trail + survives restart on either side.

## Testing model

- `test_db` fixture in `tests/conftest.py` gives each test a fresh temp SQLite DB and repoints the shared engine at it.
- Executor + MockExchange tests deterministic via `SlippageModel(rng_seed=…, noise_stdev_bps=0, reject_prob=0, latency=0)`.
- Adding a DB-touching test → depend on the `test_db` fixture, no other setup needed.
- **66 tests total** covering: comparator, HTTP rate limit / retry / circuit, screener (write + skip), executor (happy + all 4 kill-switches + stale + apr_dropped + empty book + STUCK), mock exchange, alerter (persistence + threshold + level), rebalancer (low balance + expiring + unhealthy), auth (NoAuth + Deribit OAuth caching + EIP-712 signer address), WS manager (subscribe payload + reconnect), adapters (WS ticker parsing per exchange), DeriveAuth (constants + LYRA headers + end-to-end sign+validate on testnet constants).

## Not yet wired

- Multi-account per exchange.
